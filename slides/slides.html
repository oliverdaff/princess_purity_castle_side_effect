Functor Land

Goomba problem

Defeating a Goomba

Stomp it and it turns into a coin


trait Stompable[T] {
  def stomp():T
}
case class Coin()
case class Goomba() extends Stompable[Coin] {
  def stomp() = Coin()
}

def stomp[T](s:Stompable[T]) = s.stomp()

The function stomp is our pipe, that transforms from a Stompable to a Coin.

Mario has a side kick called Luigi who made a vacuum to collect stompables


class Vacuum {
  def collect[T](s:Stompable[T]) = stomp(s)
}
          

val vacuum = new Vacuum()
val goomba = new Goomba()
vacuum.collect(goomba)

What happens when the Goomba escapes the suction?


val vacuum = new Vacuum()

vacuum.collect(null)

Solution: Check For Nulls


class Vacuum {
  def collect[T](s:Stompable[T]) = if (s == null) null else stomp(s)
}

But then the calling class runs the risk of NullPointer exceptions.

There must be a better way

Solution: Put the Goomba in a Cage


sealed trait Cage[+T]
case class FullCage[+T](value: T) extends Cage[T]
case object EmptyCage extends Cage[Nothing]

object Cage {
  def apply[T](x: T):Cage[T] = if (x == null) EmptyCage else FullCage(x)
}

class Vacuum {

  def collect[T](c:Cage[Stompable[T]]):Cage[T] = c match {
    case EmptyCage => EmptyCage
    case FullCage(s) => FullCage(stomp(s))
  }
}
          

val vac = new Vacuum()
vac.collect(Cage(Goomba()))
vac.collect(Cage(null))
